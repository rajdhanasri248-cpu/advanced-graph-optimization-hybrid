Algorithm Details

🔷 Dijkstra's Algorithm

Guarantees the shortest path by exploring nodes in order of distance. Uses a priority queue to always process the closest unvisited node.

  • Time Complexity: O((V + E) log V)
  • Space Complexity: O(V)
  • Best for: General shortest path problems

🔶 A* Algorithm

Uses heuristic values to guide the search towards the goal. More efficient than Dijkstra when good heuristics are available.

  • Time Complexity: O((V + E) log V) with good heuristic
  • Space Complexity: O(V)
  • Best for: Pathfinding with known goal position

🔯 Hybrid Algorithm

Combines A* and Dijkstra by starting with A* heuristic guidance, then switching to Dijkstra when reliability decreases.

  • Time Complexity: O((V + E) log V)
  • Space Complexity: O(V)
  • Best for: Adaptive pathfinding
